# reports-notes

This folder contains timestamped audit notes that explain:
- what changed
- why it changed
- how it was validated
- known follow-ups

## Naming Convention
- `YYYY-MM-DD-<topic>.md`
- Example: `2026-04-23-release-audit.md`

## Minimum Report Sections
- Context
- Findings
- Changes Applied
- Validation
- Follow-Ups
